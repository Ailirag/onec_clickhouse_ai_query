# 1С ClickHouse AI Assistant — Инструкция по поднятию сервера и развертыванию

Интеллектуальный веб-интерфейс на базе **React / Vite / Node.js Express**, предназначенный для анализа журналов регистрации 1С:Предприятие, экспортированных в СУБД **ClickHouse** (например, с помощью утилиты `OneSTools.EventLog`). Система автоматически транслирует естественные вопросы пользователя в оптимизированные ClickHouse SQL-запросы через AI-провайдеров (**Google Gemini** или **YandexGPT**) и визуализирует полученные результаты в интерактивные дашборды.

---

## 1. Системные требования

Перед установкой и запуском убедитесь, что на вашем сервере установлены:
* **Node.js** версии `18.x` или выше (рекомендуется LTS).
* Пакетный менеджер **npm** (поставляется вместе с Node.js) или **yarn**.
* Доступный сервер **ClickHouse** (локальный или удаленный) с HTTP-портом (обычно `8123` или `8443` для HTTPS).
* Ключи доступа к API одного из провайдеров (опционально, так как в приложении есть полнофункциональный Демо-режим):
  * **Google Gemini API Key** (может быть задан в файле `.env` как `GEMINI_API_KEY`).
  * **Yandex Cloud API-ключ** + **Folder ID** (могут быть заданы администратором интерактивно в интерфейсе).

## 2. Быстрый старт (Локальный запуск)

### Шаг 1. Клонирование и подготовка файлов
Перейдите в директорию проекта и скопируйте шаблон файла конфигурации окружения:
```bash
cp .env.example .env
```
Откройте файл `.env` в текстовом редакторе и при необходимости заполните параметры (например, `GEMINI_API_KEY` для работы с Gemini).

### Шаг 2. Установка зависимостей
Установите необходимые npm-пакеты:
```bash
npm install
```

### Шаг 3. Запуск в режиме разработки (Vite Dev Server)
Для запуска локального сервера разработки с поддержкой Hot Module Replacement (HMR) выполните:
```bash
npm run dev
```
По умолчанию приложение запустится на порту **3000** и будет доступно по адресу:
👉 `http://localhost:3000`

---

## 3. Развертывание в Production (Промышленный запуск)

Для запуска на постоянной основе в производственной среде приложение компилируется в оптимизированные статические файлы для клиента и единый бандл для Node.js сервера.

### Шаг 1. Сборка проекта
Скомпилируйте клиентский React-код и серверный скрипт:
```bash
npm run build
```
Эта команда выполнит:
1. `vite build` — компиляция React-кода в директорию `dist/`.
2. Сборку Node.js сервера (`server.ts`) в единый файл `dist/server.cjs` с помощью инструмента сборки.

### Шаг 2. Запуск готового сервера
Запустите скомпилированный сервер:
```bash
npm run start
```
Сервер запустится в Production-режиме, прослушивая порт `3000` на всех сетевых интерфейсах (`0.0.0.0`).

Для работы в фоновом режиме на Linux-сервере рекомендуется использовать менеджер процессов **PM2**:
```bash
npm install -g pm2
pm2 start dist/server.cjs --name "1c-clickhouse-ai"
pm2 save
pm2 startup
```

## 4. Структура Базы Данных ClickHouse

Для успешного анализа и автоматической генерации SQL-запросов нейросеть считывает схему базы данных. Приложение оптимизировано под структуру таблицы экспорта журналов регистрации 1С (`EventLogItems` или аналогичные).

### Ожидаемая структура таблицы `EventLogItems`:
Структура соответствует экспортёру [`OneSTools.EventLog`](https://github.com/akpaevj/OneSTools.EventLog) (поля `Event`, `Severity`, `User` и т.д.). Приложение читает реальную схему из ClickHouse, поэтому подойдёт и другая структура журнала — генерация SQL опирается на фактические имена колонок.
```sql
CREATE TABLE EventLogItems
(
    `ExporterName` LowCardinality(String),
    `FileName` LowCardinality(String),
    `EndPosition` Int64,
    `LgfEndPosition` Int64,
    `Id` Int64,
    `DateTime` DateTime('UTC'),
    `TransactionStatus` LowCardinality(String),
    `TransactionDate` DateTime('UTC'),
    `TransactionNumber` Int64,
    `UserUuid` LowCardinality(String),
    `User` LowCardinality(String),
    `Computer` LowCardinality(String),
    `Application` LowCardinality(String),
    `Connection` Int64,
    `Event` LowCardinality(String),
    `Severity` LowCardinality(String),
    `Comment` String,
    `MetadataUuid` String,
    `Metadata` LowCardinality(String),
    `Data` String,
    `DataPresentation` String,
    `Server` LowCardinality(String),
    `MainPort` Int32,
    `AddPort` Int32,
    `Session` Int64
)
ENGINE = MergeTree
PARTITION BY toYYYYMM(DateTime)
ORDER BY DateTime;
```

## 5. Две роли в интерфейсе (Многопользовательский режим)

В приложении поддерживается интерактивное разделение полномочий на две роли:

1. **Пользователь (User)**:
   * Имеет доступ к умному поисковому AI-интерфейсу.
   * Может задавать вопросы на естественном языке, получать сгенерированный SQL, выполнять его и изучать отчеты/графики.
   * **Ограничение**: Панель подключения к ClickHouse и настройки AI-провайдеров заблокированы в режиме «Только чтение» для предотвращения случайного изменения адресов баз данных или утечки конфиденциальных токенов.

2. **Администратор (Administrator)**:
   * Обладает полными правами.
   * Может переключать демо-режим, тестировать и изменять настройки СУБД ClickHouse.
   * Настраивает провайдеров искусственного интеллекта (API-ключи, идентификаторы каталогов, конкретные версии моделей YandexGPT или Gemini).
   * Пароли создаются при первом запуске и хранятся в виде PBKDF2-хеша в `passwords.json`. **Смените пароли по умолчанию сразу после установки** в разделе «Управление паролями».

## 6. Решение возможных проблем

* **Ошибка "ClickHouse connection failed"**:
  Убедитесь, что порт HTTP в ClickHouse открыт наружу (по умолчанию `8123` для HTTP, `8443` для HTTPS). Также проверьте, что в настройках указана верная база данных.
* **Ошибка "YandexGPT API Error (401)"**:
  Неверно указан API-ключ сервисного аккаунта или у сервисного аккаунта нет роли `ai.languageModels.user` в каталоге Yandex Cloud.
* **Ошибка "Folder ID не заполнен"**:
  Для корректного формирования адреса модели (`gpt://<folder_id>/yandexgpt/latest`) обязательно укажите идентификатор вашего каталога Yandex Cloud.
